# 🏗️ هيكل النظام - أنظمة التقارير (Reporting Systems)

## 📋 التعريف العام

**اسم النظام:** أنظمة التقارير (Reporting Systems)

**الهدف الأساسي:** نظام مركزي لإدارة وتحليل وعرض التقارير التشغيلية والمالية والمبيعات، مع دعم الإحصائيات، الميزانيات، وصافي الأرباح، وفق صلاحيات دقيقة متعددة المستويات.

---

## 🧩 الوحدات الرئيسية (Core Modules)

### 1. تقارير العمل الإضافي (Overtime Reports) ⚠️
- **الحالة:** نظام قائم حالياً
- **الوضع:** يعمل كما هو
- **⚠️ يمنع إجراء أي تعديل عليه**

### 2. التقارير اليومية (Daily Reports)
- تقارير مبيعات يومية
- مرتبطة بالفروع وموظفي المبيعات
- **الصفحة:** `/daily-sales-reports`

### 3. التقارير الشهرية للعقود (Monthly Contract Reports)
- عقود جديدة
- تحليل شهري حسب الفرع
- **الصفحة:** `/monthly-contract-reports`

### 4. صفحة الإحصائيات (Statistics Dashboard)
- رسوم بيانية
- مؤشرات أداء
- تجميع آلي من جميع التقارير
- **الصفحة:** `/statistics`

### 5. صفحة صافي الأرباح (Net Profit Page)
- تعتمد على:
  - الدخل
  - المصاريف
  - التصفية النهائية للربح
- **الصفحة:** `/net-profit`

### 6. صفحة الميزانية (Budget Page)
- الدخل العام
- الدخل الشهري
- فلاتر زمنية
- إضافة مصاريف
- تمهيد لاحتساب صافي الربح
- **الصفحة:** `/budget`

---

## 👥 أنواع الحسابات (User Roles)

### 1. Super Admin (السوبر أدمن)
- صلاحيات كاملة على جميع الفروع
- إدارة جميع الحسابات
- الوصول لجميع التقارير

### 2. Operation Manager (مدير أوبريشن لكل فرع)
- صلاحيات على فرعه فقط
- إدارة حسابات فرعه
- تقارير فرعه

### 3. Branch Account (حساب الفرع)
- عرض التقارير الخاصة به فقط
- بدون صلاحيات إضافة أو تعديل
- **Read-only access**

### 4. Sales Manager (مدير المبيعات حسب الفرع)
- إدارة موظفي المبيعات
- إدخال تقارير المبيعات
- إضافة عقود جديدة
- إحصائيات فرعه

### 5. Backdoor Account (حساب مخفي شامل الصلاحيات)
- يرى كل شيء
- لا يظهر في أي قوائم أو سجلات للمستخدمين
- غير مرئي في النظام
- لا يخضع لأي قيود

## 🔐 الصلاحيات والصفحات حسب كل حساب

### 1️⃣ Super Admin

**الصفحات المتاحة:**
- ✅ صفحة تقارير العمل الإضافي (من جميع الفروع)
- ✅ صفحة تقارير المبيعات (يومي + شهري)
- ✅ صفحة الإحصائيات العامة
- ✅ صفحة إدارة الحسابات
- ✅ صفحة الميزانية
- ✅ صفحة صافي الأرباح

**الصلاحيات:**
- إضافة/تعديل/تعطيل جميع الحسابات
- عرض جميع التقارير من جميع الفروع
- إضافة مصاريف عامة
- فلاتر زمنية كاملة

### 2️⃣ Operation Manager

**الصفحات المتاحة:**
- ✅ صفحة تقارير العمل الإضافي (فرعه فقط)
- ✅ صفحة إدارة الحسابات (حسابات فرعه فقط)
- ✅ صفحة الإحصائيات (فرعه فقط)

**الصلاحيات:**
- إدارة حسابات فرعه (Operation + Branch Accounts)
- عرض تقارير فرعه فقط

### 3️⃣ Sales Manager

**الصفحات المتاحة:**
- ✅ صفحة إدارة موظفي المبيعات
- ✅ صفحة إدخال تقرير مبيعات يومي
- ✅ صفحة إضافة عقد جديد
- ✅ صفحة الإحصائيات (فرعه فقط)

**الصلاحيات:**
- إضافة/تعديل موظفي المبيعات
- إدخال تقارير المبيعات اليومية
- إضافة عقود جديدة
- عرض إحصائيات فرعه

### 4️⃣ Branch Account

**الصفحات المتاحة:**
- ✅ عرض التقارير الخاصة به فقط
- ❌ بدون صلاحيات إضافة أو تعديل

**الصلاحيات:**
- **Read-only** - عرض فقط
- لا يمكنه إضافة أو تعديل أي شيء

### 5️⃣ Backdoor Account

**الصفحات المتاحة:**
- ✅ كل شيء (مثل Super Admin)
- ✅ غير مرئي في القوائم

**الصلاحيات:**
- كل شيء
- غير مرئي في النظام
- لا يظهر في logs أو قوائم المستخدمين

## 🏛️ اعتبارات هندسية ومعيارية

### 1. فصل الصلاحيات (RBAC)
- نظام صلاحيات دقيق متعدد المستويات
- كل حساب له صلاحيات محددة
- Backdoor account مخفي تماماً

### 2. قابلية التوسع (Scalability)
- تصميم قابل للتوسع
- إضافة وحدات جديدة بسهولة

### 3. توحيد مصدر البيانات (Single Source of Truth)
- قاعدة بيانات موحدة
- APIs موحدة

### 4. توافق مع أنظمة التقارير المؤسسية
- معايير تقارير موحدة
- تنسيق PDF موحد
