# ملف save.bat - حفظ تلقائي ورفع إلى GitHub

## 📋 الوظيفة

ملف `save.bat` يقوم تلقائياً بـ:
1. ✅ فحص جميع التغييرات في المشروع
2. ✅ إضافة جميع الملفات الجديدة والمعدلة
3. ✅ حفظ التغييرات مع رسالة تلقائية
4. ✅ ربط المشروع بـ GitHub (إذا لم يكن مربوطاً)
5. ✅ رفع التحديثات إلى GitHub
6. ✅ عرض رسالة نجاح مع رابط المشروع

## 🚀 طريقة الاستخدام

### الطريقة البسيطة:
```
انقر نقراً مزدوجاً على save.bat
```

### من سطر الأوامر:
```bash
save.bat
```

## ✨ المميزات

### 1. فحص تلقائي
- يتحقق من وجود Git
- يتحقق من أن المجلد هو Git repository
- يعرض جميع التغييرات قبل الحفظ

### 2. رسائل Commit ذكية
يختار رسالة commit تلقائياً بناءً على نوع التغييرات:
- تحديث واجهة المستخدم (frontend)
- تحديث Backend API
- تحديث نظام العقود
- تحديث التقارير اليومية
- تحديث تلقائي (افتراضي)

### 3. ربط تلقائي بـ GitHub
إذا لم يكن المشروع مربوطاً بـ GitHub، يسألك عن رابط الـ Repository ويربطه تلقائياً.

### 4. معالجة الأخطاء
- يعرض رسائل خطأ واضحة
- يعطي حلول للمشاكل الشائعة
- يحفظ التغييرات محلياً حتى لو فشل الرفع

## 📝 أمثلة على الاستخدام

### مثال 1: حفظ تغييرات عادية
```
1. قمت بتعديل ملف في frontend
2. شغلت save.bat
3. تم حفظ التغييرات ورفعها تلقائياً
```

### مثال 2: أول مرة (غير مربوط)
```
1. شغلت save.bat
2. طلب مني رابط GitHub
3. أدخلت الرابط
4. تم الربط والرفع تلقائياً
```

### مثال 3: بدون إنترنت
```
1. شغلت save.bat
2. تم حفظ التغييرات محلياً
3. رسالة تحذير أن الرفع فشل
4. يمكن رفعها لاحقاً عند توفر الإنترنت
```

## ⚙️ الإعدادات

### تغيير رسالة Commit الافتراضية
افتح `save.bat` وعدّل السطر:
```batch
set "COMMIT_MSG=تحديث تلقائي - %date% %time%"
```

### تغيير اسم الفرع
افتح `save.bat` وعدّل:
```batch
git push origin main
```
إلى:
```batch
git push origin master
```

## 🔧 استكشاف الأخطاء

### خطأ: "Git غير مثبت"
**الحل:** قم بتثبيت Git من https://git-scm.com/downloads

### خطأ: "هذا المجلد ليس Git repository"
**الحل:** شغّل `git init` أولاً

### خطأ: "فشل رفع التحديثات"
**الأسباب المحتملة:**
- مشكلة في الاتصال بالإنترنت
- مشكلة في المصادقة (Authentication)
- لا تملك صلاحيات الوصول

**الحلول:**
1. تحقق من الاتصال بالإنترنت
2. استخدم Personal Access Token بدلاً من كلمة المرور
3. تحقق من صلاحيات الوصول إلى الـ Repository

## 💡 نصائح

1. **استخدم save.bat بعد كل تعديل مهم** - يحافظ على نسخة احتياطية
2. **احفظ قبل إغلاق المشروع** - تأكد من حفظ جميع التغييرات
3. **راجع التغييرات قبل الحفظ** - الملف يعرض التغييرات قبل الحفظ
4. **احتفظ بنسخة محلية** - حتى لو فشل الرفع، التغييرات محفوظة محلياً
